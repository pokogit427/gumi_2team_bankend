# 10_백엔드_테스트계획

이 문서는 LocalHub 백엔드의 구현 전 테스트 기준을 정리한 문서다. 실제 기능 코드는 아직 구현하지 않으며, 이후 개발과 배포 단계의 검증 기준으로 사용한다.

---

## 1. 테스트 목표

백엔드 구현이 완료되면 다음 항목을 검증한다.

- 게시글 CRUD 동작
- 지역정보 조회와 검색 동작
- 통합 검색 동작
- 챗봇 응답 동작
- CORS 연동 동작
- 환경변수 로딩 동작
- 민감정보 노출 방지 동작
- Render 배포 후 운영 환경 동작

---

## 2. 단위 테스트 범위

단위 테스트는 다음 영역을 대상으로 한다.

- 게시글 생성 로직
- 게시글 수정 로직
- 게시글 삭제 로직
- 비밀번호 비교 로직
- 조회수 증가 로직
- 지역정보 JSON 로더 로직
- 검색 필터 로직
- 챗봇 질문 분류 로직
- 응답 정제 로직

### 권장 테스트 도구
- pytest
- FastAPI TestClient

---

## 3. API 통합 테스트 범위

API 통합 테스트는 다음 엔드포인트를 대상으로 한다.

- GET /health
- GET /api/posts
- GET /api/posts/{post_id}
- POST /api/posts
- PUT /api/posts/{post_id}
- DELETE /api/posts/{post_id}
- GET /api/locations
- GET /api/locations/{content_id}
- GET /api/search
- POST /api/chat

---

## 4. 정상 입력 테스트

### 게시글
- 제목, 내용, 비밀번호를 포함한 생성 요청이 성공한다.
- 정상적인 수정 요청이 성공한다.
- 정상적인 삭제 요청이 성공한다.

### 지역정보
- 정상적인 `query`와 `category`로 지역정보 목록 조회가 성공한다.
- 정상적인 `content_id`로 상세 조회가 성공한다.

### 검색
- `query`만 전달 시 검색 결과가 반환된다.
- `query`와 `category`를 함께 전달 시 필터링된 결과가 반환된다.

### 챗봇
- 일반적인 질문에 대해 `answer`와 `references`가 반환된다.

---

## 5. 잘못된 입력 테스트

### 요청 본문 누락
- 제목이 없는 생성 요청은 400 또는 422로 실패한다.
- 내용이 없는 생성 요청은 400 또는 422로 실패한다.
- 비밀번호가 없는 생성 요청은 400 또는 422로 실패한다.

### 잘못된 파라미터
- `page`가 음수이면 실패한다.
- `size`가 0 이하이면 실패한다.
- `category` 값이 잘못되면 빈 결과 또는 기본 처리로 처리한다.

---

## 6. 없는 데이터 테스트

- 존재하지 않는 게시글 ID로 조회 시 404를 반환한다.
- 존재하지 않는 지역정보 ID로 조회 시 404를 반환한다.
- 존재하지 않는 게시글 ID로 수정/삭제 요청 시 404를 반환한다.

---

## 7. 비밀번호 불일치 테스트

- 수정 요청 시 잘못된 비밀번호를 보내면 403을 반환한다.
- 삭제 요청 시 잘못된 비밀번호를 보내면 403을 반환한다.
- 비밀번호가 틀려도 원본 비밀번호가 응답에 노출되지 않는다.

---

## 8. JSON 결측값 테스트

- 이미지 URL이 비어 있는 항목도 정상 처리된다.
- 주소가 비어 있는 항목도 정상 처리된다.
- `mapx`, `mapy` 값이 비어 있는 경우에도 API가 예외 없이 처리된다.
- 빈 문자열을 `None` 또는 빈 값으로 처리하는 규칙이 유지된다.

---

## 9. 통합 검색 테스트

- 지역정보만 일치하는 경우 `result_type: location`으로 반환된다.
- 게시글만 일치하는 경우 `result_type: post`로 반환된다.
- 둘 다 일치하는 경우 두 결과가 함께 반환된다.
- 검색 결과가 없으면 200과 빈 `items` 배열을 반환한다.

---

## 10. 챗봇 테스트

- 관광지 추천 질문에 대하여 지역정보 결과를 포함한 응답이 반환된다.
- 음식점 위치 질문에 대해 적절한 지역정보를 포함한다.
- 게시글 검색 질문에 대해 게시글 결과를 포함한다.
- 빈 메시지 입력은 400 또는 422로 실패한다.

---

## 11. CORS 테스트

- 프론트엔드 오리진에서 API 요청이 정상적으로 허용된다.
- 허용되지 않은 Origin은 차단된다.
- preflight 요청이 정상 처리된다.

---

## 12. 환경변수 누락 테스트

- `.env`가 없으면 기본값으로 동작하는지 확인한다.
- 필수 환경변수가 비어 있으면 명확한 오류를 반환한다.
- 민감정보는 코드와 로그에 노출되지 않는다.

---

## 13. 민감정보 노출 테스트

- 비밀번호가 응답 본문에 포함되지 않는지 확인한다.
- 비밀번호가 로그에 남지 않는지 확인한다.
- API 응답, 검색 결과, 챗봇 컨텍스트에 비밀번호가 포함되지 않는지 확인한다.
- `.env` 값이 저장소에 포함되지 않는지 확인한다.

---

## 14. 배포 환경 테스트

배포 후 다음을 확인한다.

- /health가 200 응답을 반환한다.
- Swagger 문서가 접근 가능하다.
- CORS가 배포된 프론트엔드 Origin에 대해 정상 동작한다.
- SQLite 파일이 적절한 경로에 생성되거나 사용된다.
- 환경변수가 Render에서 정상 로드된다.

---

## 15. 테스트 완료 기준

다음 조건을 모두 충족하면 테스트를 완료한 것으로 판단한다.

- 주요 API에 대한 정상/오류 테스트가 모두 통과한다.
- 비밀번호 불일치와 없는 데이터 케이스가 검증되었다.
- 통합 검색과 챗봇 응답이 기대 결과를 반환한다.
- 민감정보가 노출되지 않는다.
- 배포 환경에서도 기본 API가 동작한다.

---

## 16. 테스트 우선순위

1. 게시글 CRUD
2. 지역정보 조회
3. 통합 검색
4. 챗봇 API
5. CORS 및 환경변수
6. 배포 환경 검증

---

## 17. 테스트 시 주의사항

- 실제 개인정보나 실제 사용 비밀번호를 입력하지 않는다.
- 테스트 데이터는 더미 값만 사용한다.
- 배포 테스트 시에도 민감정보를 노출하지 않는다.
- JSON 원본 데이터는 수정하지 않는다.

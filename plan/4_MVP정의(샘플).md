# 4_MVP정의(샘플)

| 열 1 | 열 2 |
| --- | --- |
| LocalHub MVP 정의서 |  |
| MVP(Minimum Viable Product)는 납기 내 고객이 요구한 핵심 가치를 검증할 수 있도록 필수 기능과 선정한 선택 기능 1개만 우선 완성하는 최소 제품 범위입니다. |  |
| 선정 권역 | 구미/경북 — 제공 JSON을 검토한 뒤 서울, 대전/충청, 구미/경북, 광주/전라, 부산 중 1개 권역으로 확정 |
| 핵심 문제 | 지역 주민과 관광객이 공공데이터의 지역정보와 실제 이용자 게시글을 한 곳에서 탐색하고 공유하기 어렵다. |
| 타깃 사용자 | 선정 권역의 관광지·맛집·축제 정보를 찾는 관광객과 회원가입 없이 지역 경험을 공유하려는 지역 주민 |
| 핵심 가치 | 로그인 없이 지역정보를 빠르게 검색하고, 공공데이터와 익명 커뮤니티 정보를 함께 확인하며, 챗봇으로 자연어 질문을 할 수 있다. |
| 성공 기준 | ① 필수 요구사항 100% 구현 ② 선정 선택 기능 1개 구현 ③ 게시글 CRUD 정상 동작 ④ 챗봇 주요 질의 4종 응답 ⑤ 360px 모바일 화면 대응 ⑥ Netlify·Render 외부 URL 정상 동작 ⑦ 민감정보 미노출 ⑧ 산출물 전체 제출 |
| 착수 전 확정 | 선정 권역, 실제 JSON 스키마·좌표·결측값, OpenAI API 사용 한도와 예산, SSAFY GitLab 제출 경로를 개발 착수 전에 확정한다. |
|  |  |
| Must have — 납품 및 검수에 반드시 포함 |  |
| 제공 JSON 기반 1개 권역 데이터 연동<br>- JSON을 불러와 관광지·맛집·축제 등 카테고리별 목록과 상세정보를 제공한다. |  |
| 익명 커뮤니티 게시판 CRUD<br>- 회원가입·로그인 없이 제목, 내용, 수정용 비밀번호로 작성·조회·수정·삭제한다. 수정·삭제는 등록 비밀번호 일치 여부만 확인한다. |  |
| 커뮤니티 데이터 영속 저장<br>- SQLAlchemy ORM과 SQLite를 사용하고, 게시글 데이터가 서버 재요청 후에도 DB에서 조회되도록 한다. |  |
| 지역정보 챗봇 API<br>- FastAPI POST /api/chat을 구현하고 관광지 추천, 축제 일정, 음식점 위치, 커뮤니티 게시글 검색 질의에 응답한다. |  |
| 챗봇 UI<br>- 플로팅 버튼, 펼친 대화창, 현재 세션의 대화 히스토리 유지, 모바일 전체 화면 표시를 구현한다. |  |
| 선정 선택 기능: 지역정보 탐색 강화 패키지<br>- 공공데이터와 커뮤니티 게시글 통합 검색, 카테고리 칩 필터, 결과 유형 구분을 구현한다. |  |
| 지정 기술 스택 준수<br>- Vue.js 3 SPA, FastAPI REST API, SQLAlchemy, SQLite로 구성하며 다른 프레임워크·DB로 대체하지 않는다. |  |
| 보안 및 AI 도구 정책 준수<br>- API 키·DB 경로·인증정보는 환경변수로 관리하고 .env를 Git에서 제외한다. VSCode Copilot과 OpenAI API만 사용한다. |  |
| 배포 및 산출물 제출<br>- Netlify와 Render에 배포하고 프론트·백엔드 저장소 URL, SQLite DB, 기능 명세서, WBS, 발표자료를 2026-07-16 15:00까지 제출한다. |  |
|  |  |
| Should have — MVP 완성도를 위해 우선 구현 |  |
| 통합 검색 범위<br>- 관광지·맛집·축제의 장소명·설명과 커뮤니티 게시글의 제목·내용을 키워드로 검색한다. |  |
| 검색 필터와 결과 표시<br>- 관광지 / 맛집 / 축제 / 커뮤니티 카테고리 칩을 제공하고 공공데이터 결과와 게시글 결과를 구분해 표시한다. |  |
| 검색 사용성<br>- 검색어와 선택 필터를 결과 화면에 유지하고, 검색 결과가 없을 때 명확한 안내 문구를 제공한다. |  |
| 데이터 신뢰 정보<br>- 제공 메타정보가 있는 경우 검색 결과 또는 상세 화면에서 출처, 라이선스, 수집일을 확인할 수 있도록 한다. |  |
| 핵심 화면 반응형 구성<br>- 홈 소개 배너, 최근 게시글, 지역정보 검색 진입, 챗봇 플로팅 버튼이 데스크톱과 모바일에서 정상 표시되도록 한다. |  |
|  |  |
| Could have — 필수·선정 기능 완료 후 여유가 있을 때 |  |
| 오늘의 지역 추천 카드<br>- 제공 JSON에서 관광지·맛집·축제를 규칙 기반으로 선정해 홈 화면에 노출한다. |  |
| 조회수·인기 게시글<br>- 상세 조회수를 집계하고 조회수 기준 인기 게시글을 홈 또는 게시판에 표시한다. |  |
| 지도 시각화<br>- 실제 JSON에 유효한 위도·경도가 충분하고 지도 API 사용 조건이 확인된 경우 관광지·맛집을 지도 핀으로 표시한다. |  |
|  |  |
| Won't have — 이번 MVP에서 명시적으로 제외 |  |
| 회원가입, 로그인, 소셜 인증, 사용자 역할·권한 체계 |  |
| 선정하지 않은 나머지 4개 권역의 동시 지원 |  |
| 댓글, 좋아요, 북마크, 이미지 첨부, 태그 등 추가 커뮤니티 기능 |  |
| 외부 날씨 API, 관광 경로 안내, WebSocket 실시간 알림, 전체 다국어 지원 |  |
| 벡터 데이터베이스·임베딩 기반 의미 검색, 사용자별 장기 채팅 기록 저장 |  |
| Vue.js·FastAPI·SQLite·Netlify·Render를 다른 기술이나 배포 플랫폼으로 변경하는 작업 |  |

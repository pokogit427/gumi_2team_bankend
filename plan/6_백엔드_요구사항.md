# 6_백엔드_요구사항

## 1. 프로젝트 목적

LocalHub 백엔드는 Vue 3 SPA 프론트엔드와 연동되는 FastAPI 기반 REST API 서버로, 다음 가치를 제공한다.

- 구미/경북권 공공데이터 JSON 기반 지역정보 조회
- 익명 커뮤니티 게시글 CRUD
- 공공데이터와 게시글의 통합 검색
- 챗봇 질의 응답
- 프론트엔드와의 CORS 연동 및 배포 환경 지원

> 가정: 백엔드의 기본 역할은 데이터 제공, 저장, 검색, 챗봇 응답 생성이며, 사용자 인증이나 권한 관리 기능은 포함하지 않는다.

---

## 2. 백엔드 담당 범위

백엔드는 다음 범위를 담당한다.

- FastAPI 서버 구성
- SQLite + SQLAlchemy 기반 게시글 저장
- data 폴더 JSON 로딩 및 메모리 캐싱
- 지역정보 목록/상세/검색 API 제공
- 익명 게시판 CRUD API 제공
- POST /api/chat 챗봇 API 제공
- CORS 설정 및 환경변수 기반 설정 관리
- 테스트 코드와 배포 준비

---

## 3. 필수 기능

### 3.1 서버 상태 확인
- GET /health 또는 유사 엔드포인트로 서버 상태 확인 가능해야 한다.
- 정상 응답 시 서버가 실행 중임을 반환한다.

### 3.2 익명 게시판 CRUD
- 목록 조회, 상세 조회, 생성, 수정, 삭제 API를 제공한다.
- 게시글은 제목, 내용, 수정용 비밀번호를 포함한다.
- 수정/삭제 시 입력 비밀번호와 저장 비밀번호를 평문 비교한다.
- 비밀번호는 API 응답, 로그, 검색 결과, 챗봇 컨텍스트에 포함하지 않는다.

### 3.3 지역정보 조회
- data 폴더의 JSON 파일을 읽어 지역정보 목록과 상세 정보를 제공한다.
- 각 데이터는 JSON 원본을 수정하지 않고 메모리에 캐싱한다.
- 카테고리별 조회, 검색, 상세 조회를 지원한다.

### 3.4 통합 검색
- 공공데이터(JSON)와 커뮤니티 게시글을 함께 검색할 수 있어야 한다.
- 검색 결과는 데이터 소스 구분(공공데이터/게시글)을 명시해야 한다.

### 3.5 챗봇 API
- POST /api/chat 엔드포인트를 제공한다.
- 사용자 질문을 기준으로 지역정보 또는 게시글 관련 응답을 생성한다.
- 질문의 예시는 관광지 추천, 축제 일정, 음식점 위치, 게시글 검색 등으로 제한한다.

### 3.6 CORS 및 프론트 연동
- 프론트엔드의 요청을 허용하기 위해 CORS 설정을 제공한다.
- 배포 환경에서도 프론트와 백엔드가 연동될 수 있어야 한다.

## 4. 선택 기능

다음 선택 기능은 구현 우선순위가 낮으며, 필수 기능이 완료된 이후 검토한다.

- 공공데이터 + 게시글 통합 검색 강화
- 카테고리 칩 필터 지원
- 결과 유형 구분(공공데이터/게시글)
- 출처, 라이선스, 수집일 표시

> 가정: 선택 기능은 백엔드 API 응답에 메타정보를 포함하는 수준으로 구현한다.

## 5. 제외 기능

다음 기능은 이번 백엔드 요구사항 범위에서 제외한다.

- 회원가입, 로그인, 사용자 인증
- 댓글, 좋아요, 북마크
- 이미지 업로드
- 외부 API 연동(예: OpenAI, 지도, 날씨)
- 다국어 지원
- 실시간 알림
- 사용자별 세션/채팅 기록 저장

## 7. 제약사항

- 지역정보는 data 폴더의 JSON 파일을 읽어 제공한다.
- JSON 원본은 수정하지 않는다.
- 게시글만 SQLite에 저장한다.
- 회원가입과 로그인은 구현하지 않는다.
- 비밀번호는 평문으로 저장·비교한다.
- 비밀번호는 응답에 포함하지 않는다.
- API 키와 환경설정은 .env로 관리한다.
- .env는 Git에 포함하지 않는다.
- 요청하지 않은 기능은 임의로 추가하지 않는다.
- 기존 정상 코드를 함부로 삭제하거나 전체 재작성하지 않는다.

### 8.3 챗봇 입력
- POST /api/chat 요청 body에 `message` 필드가 포함되어야 한다.

## 10. 데이터 저장 정책

- 지역정보는 JSON 파일을 읽어 메모리에 캐시한다.
- 게시글은 SQLite 데이터베이스에 저장한다.
- 게시글 저장 시 테이블은 `posts`로 관리한다.
- JSON 원본 파일은 변경하지 않는다.
- 서버 재시작 시에도 게시글은 SQLite DB를 통해 유지되어야 한다.

> 가정: 지역정보 캐시는 애플리케이션 시작 시 한 번 로드하고, 이후 메모리에 유지한다.

---

## 11. 비밀번호 처리 정책

- 게시글 작성/수정/삭제 요청 시 비밀번호를 받는다.
- 비밀번호는 평문으로 저장한다.
- 비밀번호는 API 응답에 포함하지 않는다.
- 비밀번호는 로그, 검색 결과, 챗봇 컨텍스트에 포함하지 않는다.
- 수정·삭제 시 비밀번호 불일치 시 400 또는 403 수준의 오류를 반환한다.

---

## 12. 환경변수 관리 정책

- API 키, DB 경로, CORS 허용 origin 등은 `.env` 파일로 관리한다.
- `.env`는 Git에 포함하지 않는다.
- 환경변수는 로컬 실행과 Render 배포 환경 모두에서 사용 가능해야 한다.

---

## 13. 지역정보 JSON 처리 정책

- JSON 파일은 직접 수정하지 않는다.
- 파일 목록을 순회해 `items` 배열을 로드한다.
- `mapx`, `mapy`는 문자열이므로 백엔드에서 float로 변환한다.
- 빈 문자열이 있는 필드는 None 또는 빈 값으로 처리한다.
- 카테고리는 `contenttypeid` 기준으로 분류한다.
- 데이터 로딩은 애플리케이션 시작 시 한 번 수행하고 캐시한다.

---

## 14. 챗봇 처리 정책

- 챗봇은 사용자 질문을 단순 분류하여 응답한다.
- 질문 유형은 다음과 같이 제한한다.
  - 관광지 추천
  - 축제 일정
  - 음식점 위치
  - 게시글 검색
- 응답은 지역정보 또는 게시글 검색 결과를 기반으로 구성한다.
- 외부 AI 모델 호출은 구현 범위에서 제외한다.

> 가정: 챗봇은 규칙 기반 응답으로 구현한다.

---

## 15. 오류 처리 기준

- 필수 입력값 누락 시 400 Bad Request를 반환한다.
- 존재하지 않는 리소스 요청 시 404 Not Found를 반환한다.
- 비밀번호 불일치 시 400 또는 403을 반환한다.
- JSON 로딩 실패 시 500 Internal Server Error를 반환한다.
- 예외 발생 시 응답 body에 오류 메시지를 포함한다.

## 17. 요구사항별 검증 방법

| 항목 | 검증 방법 |
|---|---|
| 서버 상태 확인 | GET /health 호출 후 200 응답 확인 |
| 게시글 생성 | POST /api/posts 호출 후 DB 저장 확인 |
| 게시글 수정/삭제 | PUT/DELETE 요청 후 비밀번호 검증 확인 |
| 지역정보 목록 | GET /api/locations 호출 후 JSON 데이터 반환 확인 |
| 지역정보 상세 | GET /api/locations/{id} 호출 후 정보 반환 확인 |
| 통합 검색 | query 파라미터로 검색 후 공공데이터/게시글 결과 확인 |
| 챗봇 API | POST /api/chat 호출 후 응답 메시지 확인 |
| CORS | 프론트 origin에서 요청 시 정상 응답 확인 |
| 환경변수 | .env 로드 여부 확인 |
| 데이터 무결성 | JSON 원본 수정 없이 로딩되는지 확인 |

# LocalHub 프로젝트 개요

## 1. 프로젝트 목적
LocalHub는 지역 주민과 관광객이 회원가입 없이 익명으로 지역 정보를 공유하고 소비할 수 있는 웹 서비스입니다. 주어진 과제에서는 `구미/경북권`의 공공데이터를 기반으로 지역 정보 탐색과 익명 게시판, 그리고 챗봇 질의 응답을 하나의 서비스로 통합합니다.

## 2. 선정 권역 및 데이터
- 권역: `구미/경북권`
- 제공 데이터: 한국관광공사 TourAPI 4.0 기반 JSON 파일
- 포함 카테고리:
  - 관광지
  - 음식점
  - 축제공연행사
  - 숙박
  - 쇼핑
  - 문화시설
  - 레포츠
  - 여행코스
- 출처 및 라이선스: `한국관광공사`, `공공누리 제3유형`
- 데이터 출처 문서: `data/SOURCE.md`
- 데이터 스키마 문서: `data/SCHEMA.md`

## 3. 요구사항 요약
### 필수 요구사항
- `Vue.js 3` 기반 SPA 프론트엔드
- `FastAPI` 기반 REST API 백엔드
- `SQLite` + `SQLAlchemy` 데이터베이스
- 회원가입·로그인 없는 익명 커뮤니티 게시판 CRUD
- 게시글 작성 시 제목, 내용, 수정용 비밀번호 입력
- 수정/삭제 시 평문 비밀번호 비교
- 제공 JSON 기반 지역 정보 연동
- `POST /api/chat` 챗봇 API 엔드포인트 구현
- 챗봇은 공공 데이터 기반 자연어 질의에 응답
- 챗봇 주요 질의: 관광지 추천, 축제 일정, 음식점 위치, 게시글 검색
- 챗봇 UI: 대화 히스토리 유지, 플로팅 버튼, 모바일 대응
- Netlify + Render 배포

### 선택 요구사항
- 최소 1개 선택 기능 구현
- 추천 선택 기능: `지역정보 탐색 강화 패키지`
  - 공공데이터+게시글 통합 검색
  - 카테고리 칩 필터
  - 결과 유형 구분
  - 출처/라이선스/수집일 표시

## 4. MVP 방향성
### 핵심 가치
- 로그인 없이 지역 정보를 빠르게 찾고
- 공공데이터와 커뮤니티 정보를 함께 활용하며
- 챗봇으로 자연어 질의를 지원하는 서비스

### MVP 포함 항목
- 제공 JSON 기반 지역 정보 목록/상세 화면
- 익명 커뮤니티 게시판 CRUD
- 게시글 데이터 DB 저장 및 영속성 보장
- 통합 검색 + 카테고리 필터
- 챗봇 API + UI
- 반응형 모바일 화면
- 배포 URL 정상 동작

### 우선순위 기능
1. `권역 확정 및 JSON 품질 검토`
2. `DB 스키마 및 API 설계`
3. `게시판 CRUD 구현`
4. `지역 정보 목록/검색 구현`
5. `챗봇 응답 로직 구축`
6. `프론트 UI 및 모바일 대응`
7. `배포 및 외부 URL 검증`

## 5. 기술 스택
- 프론트엔드: `Vue.js 3`, SPA, `Vue Router`
- 백엔드: `FastAPI`
- 데이터베이스: `SQLite`, `SQLAlchemy`
- 챗봇: FastAPI `POST /api/chat` 엔드포인트
- 배포: `Netlify` (프론트), `Render` (백엔드)
- 환경변수 관리: `.env` 및 Git 무시

## 6. 주요 제약 조건
- 선택 권역은 `구미/경북권` 단일 권역
- 커뮤니티 게시글은 반드시 DB에 저장
- 비밀번호는 평문으로 저장·비교
- 개인정보 수집 기능은 추가하지 않음
- 민감 정보는 절대 소스코드에 포함하지 않음
- OpenAI API 사용은 허용 범위 내에서만
- 납기: `2026-07-16 15:00`
- 제출물: 프론트·백엔드 저장소 URL, SQLite DB, 배포 URL, 기능 명세서, WBS, 발표 자료

## 7. 데이터 특징 및 고려 사항
- JSON 데이터는 `TourAPI 4.0` 원본 형식
- 좌표 필드(`mapx`, `mapy`)는 문자열로 제공되므로 숫자 변환 필요
- 일부 필드가 빈 문자열일 수 있음(이미지, 주소 등)
- 카테고리 구분은 `contenttypeid`로 가능
- 제공 데이터는 변경 금지 라이선스이므로 원본 텍스트를 그대로 활용

## 8. 프로젝트 성공 기준

# LocalHub 백엔드 후속 작업 프롬프트

## 사용 범위

- 현재 완료 단계: `17단계 — 챗봇 검색 로직 및 POST /api/chat 구현`
- 이어서 수행할 단계: `18단계 ~ 26단계`
- 수행 담당: 후속 작업 담당자
- 기준 문서:
  - `plan/` 폴더의 프로젝트 기획 문서
  - `data/SCHEMA.md`
  - `data/SOURCE.md`
  - `COMMIT_CONVENTION.md`
  - `시퀀스다이어그램.md`
  - 현재 구현된 `backend/` 코드

## 공통 작업 규칙

각 단계의 프롬프트를 VS Code GitHub Copilot Chat에 순서대로 입력합니다.

- 이전 단계의 정상 기능을 임의로 삭제하지 않습니다.
- 한 단계에서 요청한 작업만 수행합니다.
- 실제 구현 결과와 문서가 다르면 실제 구현을 기준으로 문서를 갱신합니다.
- API 키, 비밀번호, `.env` 값은 코드·로그·문서에 노출하지 않습니다.
- 각 단계가 끝나면 관련 테스트와 전체 테스트를 실행합니다.
- `COMMIT_CONVENTION.md` 규칙에 맞는 영어 커밋 메시지를 사용합니다.
- 테스트하지 않은 기능을 완료로 표시하지 않습니다.

# 18단계. CORS 및 프론트엔드 연동 설정

```text
현재 FastAPI 앱과 plan/7_API_명세서.md를 확인해줘.

이번 단계에서는 Vue 프론트엔드 연동을 위한 CORS와 API 공통 설정만 구현해줘.

구현 요구사항:
1. ALLOWED_ORIGINS 환경변수를 추가한다.
2. 여러 Origin을 쉼표로 구분할 수 있게 한다.
3. 로컬 Vue 기본 주소를 .env.example에 예시로 넣는다.

5. 전체 Origin을 무조건 "*"로 허용하지 않는다.
6. GET, POST, PUT, DELETE, OPTIONS를 처리한다.
7. 필요한 Header를 허용한다.
8. 이 프로젝트는 로그인 쿠키를 사용하지 않으므로 allow_credentials 설정을 적절하게 결정하고 이유를 주석 또는 문서에 남긴다.
9. 기존 API 동작을 변경하지 않는다.
10. 환경변수 값을 로그에 출력하지 않는다.

추가로 다음 파일을 생성해줘.

작업 후 CORS 테스트 결과, 변경 파일, 커밋 메시지를 알려줘.
```
